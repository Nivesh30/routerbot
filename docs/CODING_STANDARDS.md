## Project Structure Rules

### Module Boundaries

```
src/routerbot/
├── core/       # ZERO external deps beyond stdlib + pydantic. No FastAPI, no DB, no Redis.
├── providers/  # Depends on core/ only. Each provider is self-contained.
├── router/     # Depends on core/ and providers/. No DB, no HTTP framework.
├── proxy/      # Depends on everything above. FastAPI lives here and ONLY here.
├── auth/       # Depends on core/ and db/. Auth logic independent of HTTP.
├── db/         # Depends on core/ only. SQLAlchemy models and repositories.
├── cache/      # Depends on core/ only. Cache backends.
├── observability/ # Depends on core/ only. Callback integrations.
└── utils/      # Pure utility functions. ZERO business logic.
```

### Import Rules

1. **No circular imports.** CI will detect and fail on circular imports.
2. **No wildcard imports** (`from module import *` is banned).
3. **No relative imports beyond one level** (use `from routerbot.core.types import ...`).
4. **Module dependency direction must be strictly downward** — `proxy/` can import from `core/` but `core/` cannot import from `proxy/`.
5. **Provider modules must not import from each other** — `openai/` cannot import from `anthropic/`.

### File Naming

- All Python files: `snake_case.py`
- All TypeScript files: `camelCase.tsx` (components) or `camelCase.ts` (utilities)
- Test files: `test_<module_name>.py`
- Migration files: Auto-generated by Alembic (timestamped)
- Config files: `snake_case.yaml` / `snake_case.json`

### Docstrings

Use **Google-style** docstrings on all public functions, classes, and modules:

```python
def get_completion(
    model: str,
    messages: list[Message],
    *,
    stream: bool = False,
    temperature: float | None = None,
) -> ModelResponse:
    """Get a chat completion from the specified model.

    Translates the request to the provider's format, sends it,
    and normalizes the response to OpenAI format.

    Args:
        model: Model identifier in format "provider/model_name".
        messages: List of chat messages.
        stream: Whether to stream the response.
        temperature: Sampling temperature (0.0 to 2.0).

    Returns:
        Normalized model response in OpenAI format.

    Raises:
        AuthenticationError: If the provider API key is invalid.
        RateLimitError: If the provider rate limit is exceeded.
        ModelNotFoundError: If the specified model doesn't exist.
    """
```

### Class Design

```python
from dataclasses import dataclass, field
from pydantic import BaseModel, Field

# Use Pydantic BaseModel for API request/response schemas
class CompletionRequest(BaseModel):
    model: str
    messages: list[Message]
    stream: bool = False
    temperature: float | None = None

    model_config = ConfigDict(extra="allow")  # Allow provider-specific params

# Use dataclasses for internal data structures
@dataclass(frozen=True, slots=True)
class ProviderConfig:
    name: str
    api_base: str
    api_key: str
    max_retries: int = 3

# Use Protocol for interfaces (duck typing)
from typing import Protocol

class CacheBackend(Protocol):
    async def get(self, key: str) -> str | None: ...
    async def set(self, key: str, value: str, ttl: int | None = None) -> None: ...
    async def delete(self, key: str) -> None: ...
```

## Type Safety

### MyPy Configuration

```toml
[tool.mypy]
python_version = "3.11"
strict = true
warn_return_any = true
warn_unused_configs = true
disallow_untyped_defs = true
disallow_incomplete_defs = true
check_untyped_defs = true
disallow_untyped_decorators = true
no_implicit_optional = true
warn_redundant_casts = true
warn_unused_ignores = true
show_error_codes = true

[[tool.mypy.overrides]]
module = "tests.*"
disallow_untyped_defs = false
```

### Rules

1. **All function signatures must be fully typed** — no `Any` without explicit justification.
2. **Use `|` union syntax** instead of `Optional` or `Union` (Python 3.11+ style).
3. **Use `list[X]`, `dict[K, V]`** instead of `List[X]`, `Dict[K, V]`.
4. **Prefer `TypedDict` for dict-like structures with known keys.**
5. **Use `Literal` for string enums when the set is small** (< 5 values).
6. **Use `Enum` or `StrEnum` for larger enumerated sets.**
7. **All Pydantic models must have explicit field types.**

```python
# ✅ Good
def process(data: dict[str, str | int]) -> list[Result]: ...

# ❌ Bad
def process(data: dict) -> list: ...
```

## Error Handling

### Exception Hierarchy

```python
# All RouterBot exceptions inherit from a common base
class RouterBotError(Exception):
    """Base exception for all RouterBot errors."""
    status_code: int = 500
    message: str

class AuthenticationError(RouterBotError):
    status_code = 401

class PermissionDeniedError(RouterBotError):
    status_code = 403

class NotFoundError(RouterBotError):
    status_code = 404

class RateLimitError(RouterBotError):
    status_code = 429

class BadRequestError(RouterBotError):
    status_code = 400

class ProviderError(RouterBotError):
    """Wraps upstream provider errors."""
    provider: str
    original_error: Exception | None = None

class ConfigurationError(RouterBotError):
    status_code = 500
```

### Rules

1. **Never catch bare `Exception`** — always catch specific types.
2. **Always re-raise or log** — no silent `except: pass`.
3. **Map all provider exceptions** to the RouterBot exception hierarchy.
4. **Include context** in error messages (model, provider, request_id).
5. **Use structured logging** (JSON format) for all error logs.
6. **Never expose internal errors to clients** — map to OpenAI-compatible error format.

```python
# ✅ Good
try:
    response = await provider.complete(request)
except httpx.TimeoutException as e:
    logger.warning("Provider timeout", extra={"provider": provider.name, "model": model})
    raise ProviderError(
        message=f"Provider {provider.name} timed out",
        provider=provider.name,
        original_error=e,
    ) from e

# ❌ Bad
try:
    response = await provider.complete(request)
except Exception:
    pass
```

## Async/Await Patterns

### Rules

1. **All I/O operations must be async** — HTTP calls, DB queries, Redis ops, file I/O.
2. **Use `httpx.AsyncClient`** for HTTP requests (not `requests`).
3. **Use connection pooling** — create clients at startup, share across requests.
4. **Use `asyncio.gather()` for concurrent operations** where safe (parallel provider calls).
5. **Always set timeouts** on all external calls.
6. **Provide sync wrappers** (`completion()`) that call async internally for SDK user convenience.

```python
# ✅ Good: Connection pool shared across requests
class OpenAIProvider:
    def __init__(self, config: ProviderConfig) -> None:
        self._client = httpx.AsyncClient(
            base_url=config.api_base,
            timeout=httpx.Timeout(connect=5.0, read=60.0, write=10.0, pool=10.0),
            limits=httpx.Limits(max_connections=100, max_keepalive_connections=20),
        )

    async def complete(self, request: CompletionRequest) -> ModelResponse:
        response = await self._client.post("/chat/completions", json=request.model_dump())
        response.raise_for_status()
        return ModelResponse.model_validate(response.json())

    async def close(self) -> None:
        await self._client.aclose()
```

## Database Patterns

### Rules

1. **Repository pattern** — all DB access goes through repository classes, never raw queries in routes.
2. **Async sessions** — use `AsyncSession` from SQLAlchemy.
3. **Transaction management** — use context managers for transaction boundaries.
4. **Migrations are mandatory** — every schema change requires an Alembic migration.
5. **No raw SQL in application code** — use SQLAlchemy ORM/Core expressions.
6. **Index all foreign keys** and columns used in `WHERE`/`ORDER BY` clauses.
7. **Use UUIDs for primary keys** (not auto-increment integers).

```python
# Repository pattern
class KeyRepository:
    def __init__(self, session: AsyncSession) -> None:
        self._session = session

    async def get_by_hash(self, key_hash: str) -> VirtualKey | None:
        result = await self._session.execute(
            select(VirtualKey).where(VirtualKey.key_hash == key_hash)
        )
        return result.scalar_one_or_none()

    async def create(self, key: VirtualKey) -> VirtualKey:
        self._session.add(key)
        await self._session.flush()
        return key
```

## API Design

### Rules

1. **All public endpoints must be OpenAI-compatible** — same request/response format.
2. **All management endpoints use JSON request/response.**
3. **Version prefix**: `/v1/` for LLM endpoints, no prefix for management endpoints.
4. **Use proper HTTP status codes** (200, 201, 400, 401, 403, 404, 429, 500).
5. **All responses must include `request_id`** in headers.
6. **Pagination**: Use cursor-based pagination for list endpoints.
7. **Rate limit headers**: Include `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`.

```python
# Route definition pattern
@router.post("/v1/chat/completions", response_model=CompletionResponse)
async def chat_completions(
    request: CompletionRequest,
    auth: AuthContext = Depends(verify_auth),
    rate_limiter: RateLimiter = Depends(get_rate_limiter),
) -> CompletionResponse:
    """Create a chat completion."""
    await rate_limiter.check(auth.key_id)
    response = await router_instance.complete(request, auth=auth)
    return response
```

### Test Rules

1. **Every new feature must include tests.** No exceptions.
2. **Every bug fix must include a regression test.**
3. **Use `pytest` with `pytest-asyncio`** for async tests.
4. **Mock all external HTTP calls** — use `respx` or `httpx` mocking.
5. **Use factories for test data** — no raw dict construction in tests.
6. **Test names must describe the scenario**: `test_completion_returns_error_when_api_key_invalid`
7. **Mark slow tests** with `@pytest.mark.slow`.
8. **Integration tests use test containers** — PostgreSQL and Redis via Docker.

```python
# Test example
import pytest

@pytest.mark.asyncio

    with pytest.raises(AuthenticationError):
        await acompletion(
            model="openai/gpt-4o",
            messages=[{"role": "user", "content": "Hello"}],
        )
```

## Security Rules

1. **Never log API keys, tokens, or secrets** — even in debug mode.
2. **Hash all stored API keys** with SHA-256 + salt.
3. **Validate all input** — use Pydantic models, reject unexpected fields in strict mode where needed.
4. **Set Content Security Policy headers** on dashboard.
5. **Rate limit all endpoints** — including management endpoints.
6. **CORS must be explicitly configured** — no wildcard in production.
7. **Use `secrets.token_urlsafe()`** for key generation.
8. **All passwords/keys in config must support secret manager references** (`os.environ/`, `aws_secret/`, etc.).
9. **SQL injection prevention** — use parameterized queries only (ORM handles this).
10. **No `eval()`, `exec()`, or `__import__()`** anywhere in the codebase.
11. **Audit log all admin actions** — key creation, deletion, config changes.
12. **Set HTTPS-only in production** — enforce via middleware.

## Performance Standards

1. **P95 latency overhead < 10ms** — RouterBot should add minimal latency over direct provider calls.
2. **Support 1000+ RPS** per instance with connection pooling.
3. **Streaming must start within 100ms** of provider first token.
4. **Database queries must use indexes** — no full table scans in hot paths.
5. **Cache hot data** — model configs, key lookups, rate limit counters.
6. **Connection pooling** — reuse HTTP connections to providers, DB connections, Redis connections.
7. **No blocking I/O on the event loop** — use `run_in_executor()` for any unavoidable sync I/O.
8. **Profile before optimizing** — use `py-spy` or `cProfile`, don't guess.
